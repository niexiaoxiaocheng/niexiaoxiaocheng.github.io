✨大厂的远程调试工具

  • 阿里巴巴的arthas 推荐指数 :star::star::star::star::star:

    Arthas 是Alibaba开源的Java诊断工具，支持JDK6+,支持Linux/Mac/Windows ,命令行交互，支持Tab补全，骚操作很多 例如：动态修改日志级别（可精确到类），热更新代码

    对生产环境影响极小。

  • 唯品会的vjtools 推荐指数 :star:

    VJTools是唯品会在Java社区积累和产出的整理，主要从规范、源码以及排查问题工具上帮助Java工程人员提高效率和避免踩坑

  • 去哪儿网的bistoury 推荐指数 :star::star::star:

    Bistoury是去哪儿网基于自己公司内部agent的基础上集成Arthas和Vjtools，页面相对友好，功能也更加丰富，例如浏览器访问bistoury可以查看反编译项目的代码（模拟IDEA调试），通过web界面提供断点调试的功能，可以在不阻塞应用的情况下捕获断点处的信息（包括本地变量、成员变量、静态变量和方法调用栈）。

    Bistoury提供了日志查看功能，可以使用tail、grep等命令对单台或同时对多台机器的日志进行查看。

    Bistoury提供可视化页面实时查看机器和应用的各种信息，包括主机内存和磁盘使用、cpu使用率和load、系统配置文件、jar包信息，jvm信息、内存使用和gc等等。

    …

  • 比较

  • Arthas和vjtools通过命令行或类似的方式使用，不可否认命令行在很多时候具有比较高的效率；但图形化界面也有其自身的优点，特别是在参数复杂时使用起来更加简单，效率更高。

    Bistoury在保留命令行界面的基础上，还对很多命令提供了图形化界面，方面用户使用。

    Arthas和vjtools针对单台机器，从机器的维度对系统进行诊断，没有提供全局的视角；而在线应用往往部署在多台机器，

    Bistoury可以和使用方应用中心整合，从应用的维度对系统进行诊断，提供了更多的可能。

    Arthas和vjtools在使用上，要么登录机器，要么需要使用者提供相应的ip和端口；

    Bistoury去掉各种设置，提供统一的web入口，从页面上选择应用和机器即可使用。

    但是：Arthas相对来说使用人数最多，使用手册最详细，并且是在持续更新和完善的。

IDEA 好用的插件

  • Alibaba P3C编码规约扫描工具

    帮你找出不符合阿里巴巴编程规约上规定的编码问题

  • Sonal Lint 插件

    研发必备，集成之后 可以使用默认的扫描规则，也可以连接公司的http://sonar.thunisoft.com/服务器，用公司的代码规则来分析代码质量问题，做到写一行检查一行，再也不用等项目开发完再统一修改构建问题啦。

  • Translation 插件

    集成了 谷歌翻译、有道翻译、百度翻译 阅读源代码注释不用Ctrl+C,Ctrl+V了，鼠标右键选中即可。

  • Mybatis Log 插件

    适合没集成SqlFx的老项目使用，能根据PSQL 和参数，打印出完整的SQL、执行方法、请求名、执行时间。排查问题是很方便。

    同时还可以实现Mapper和Xml之间的快速跳转。

  • CodeGlance 代码迷你缩放图插件

    跟Vscode 差不多，现在每个类 内容不会太多 用不到了。

  • Json Parser json 内置的Json 格式化插件

    utools 里面有，更方便 还能支持XML YAML 粘贴进去 自动转JSON

  • Lombok

    配合lombak.jar包 使用 通过注解来简化一些比较臃肿的代码

  • leetcode editor

    刷题必备,支持leetcode.com和leetcode-cn.com,可以同步LeetCode上的题目，刷题、测试、提交、查看优秀解答。
