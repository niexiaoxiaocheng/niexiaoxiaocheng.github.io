Java远程调试

一、前言

你是否碰到过一下问题？

  • 1.生产环境上出现问题了，但是服务器上没有源码，也没有IDEA 怎么办？
  • 2.为什么我的代码本地跑可以，打包放在服务器上就不行了？
  • 测试环境/生产环境中我想看下这个对象里面存着哪些信息，为什么会报错？
  • 定位问题疲于打log日志，才能知道各个阶段变量的值？
  • 。。。。

通过远程调试你可以用本地的开发工具对远端正在运行的程序进行debug调试。

二、Java远程调试是什么

JDWP（Java Debug Wire Protocol）

Java远程调试的原理是两个VM之间通过debug协议进行通信，然后达到远程调试的目的。两者之间可以通过socket进行通信。首先被debug程序的虚拟机在启动时要开启debug模式，启动debug监听程序。

三、怎么开启远程调试

1.Tomcat中间件：

Window：编辑bin目录下的startup.bat，在脚本前面添加下面配置：

### 下面需要保持一行
SET CATALINA_OPTS= -Xdebug -Xrunjdwp:transport=dt_socket,server=y,suspend=n,address=8787

Linux:在bin目录下catalina.sh中的首行添加：

CATALINA_OPTS=-Xdebug  -Xrunjdwp:transport=dt_socket,server=y,suspend=n,address=8787

2.SpringBoot 项目

在java -jar之间添加

-Xdebug -Xrunjdwp:transport=dt_socket,address=8787,server=y,suspend=n

如下图：监听8787端口

四、使用IDEA进行远程调试

1.打开Intellij IDEA，在顶部靠右的地方选择”Edit Configurations…”，进去之后点击+号，选择【Remote】,按照下图的指示填写红框内容

Name：名称，可以随便填，无实际功能。

Host: 被调试程序部署的机器ip/hostname

Port:为第三步设置的address 端口

2.设置完毕后点击Appay,最后点击OK即可。

3.本地IDEA启动debug模式。

4.观察控制台日志，出现Connected to the target VM, address: '172.26.19.77:8787', transport: 'socket',则启动成功。

5.接下来就可以在源码上打断点，程序运行到断点处时，会被断住。进行调试即可。

五、参数解释

  • -XDebug: 启用调试；

  • -Xrunjdwp :加载JDWP的JPDA参考执行实例；

  • transport: 用于在调试程序和VM使用的进程之间通讯；

  • dt_socket: 套接字传输;

  • server=y/n:VM是否需要作为调试服务器执行；

  • address=8787:调试服务器监听的端口号,端口号可自行定义，未被占用即可。

  • suspend=y/n: 是否在调试客户端建立连接之后启动 VM,如果为y,则在启动项目会阻塞，等待调试器连接，才可继续往下运行；
